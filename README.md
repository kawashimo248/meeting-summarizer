# AI 議事録ジェネレーター (スマホ対応)

スマートフォンやPCで会議音声を録音・アップロードし、Google の Gemini API のみ（無料枠あり）を使用して、自動的に「音声文字起こし」と「構造化された要約議事録」を作成するWebアプリケーションです。

サーバー不要の静的フロントエンド設計（GitHub Pages対応）となっており、APIキーは利用者自身の端末（ブラウザのLocalStorage）にのみ安全に保存されます。

---

## 主な機能

- 📱 **モバイル最適化（スマホ対応）**: タッチ操作しやすい録音UIとレスポンシブな画面設計。
- 🎙️ **マイク録音**: ブラウザから直接録音可能。録音状況を視覚的に伝えるリアルタイム波形ビジュアライザー搭載。
- 📁 **ファイルアップロード**: スマホやPCで録音した既存ファイル（MP3, M4A, WAV, WEBM等）の読み込み。
- 🤖 **Gemini 1.5 / 3.5 ネイティブ音声解析**: AIが音声を直接聴いて、高精度な「文字起こし」と「要約」を1回で同時に生成。
- 📝 **プロンプトテンプレート**: 「標準的な議事録」「ブレインストーミング」「ToDo特化」などの切り替え機能。
- 💾 **結果エクスポート**: ワンタップで要約Markdownのコピーやファイル保存が可能。
- 🧪 **デモシミュレーション機能**: APIキーをお持ちでない場合でも、アプリの動作や生成される議事録イメージを体験可能。

## 導入手順 (GitHub Pages へのデプロイ)

本アプリはHTML、CSS、JavaScriptの静的ファイルのみで構成されているため、GitHub Pagesを使って即座に一般公開・アクセス可能にすることができます。

### 1. 新しいGitHubリポジトリの作成
1. [GitHub](https://github.com/) にログインし、新しいリポジトリ（例: `meeting-summarizer`）を作成します。
2. 作成したリポジトリに、本フォルダ（`J.議事録自動作成アプリ`）内の全ファイル (`index.html`, `styles.css`, `app.js`) をプッシュします。

### 2. GitHub Pagesの設定
1. リポジトリの画面上部にある **Settings**（設定）タブを開きます。
2. 左メニューから **Pages** を選択します。
3. **Build and deployment** セクションで以下を設定します：
   - **Source**: `Deploy from a branch`
   - **Branch**: `main` (または `master`)、フォルダは `/ (root)` を選択。
4. **Save** ボタンをクリックします。

### 3. 公開確認
1. 数分待つと、画面上部に公開URL（例: `https://[ユーザー名].github.io/meeting-summarizer/`）が表示されます。
2. お持ちのスマートフォン（SafariやChrome）やPCからそのURLを開くことで、アプリが使用可能になります。

## 使い方

1. **Gemini APIキーの設定**:
   - 初回起動時、画面右上にある ⚙️（歯車アイコン）をタップします。
   - ご自身の「Gemini APIキー」を入力し、保存します。
   - **キーの取得先**: [Google AI Studio](https://aistudio.google.com/app/apikey) から無料で即座に取得できます。
   - ※ キーはブラウザ内にのみ保存され、開発者を含むいかなる外部サーバーにも送信されません。

2. **録音・ファイルの準備**:
   - **その場で録音する場合**: 「マイク録音」タブで赤いマイクボタンを押すと録音が開始されます。一時停止や停止も可能です。
   - **録音済みファイルを使う場合**: 「ファイル選択」タブでスマホ内にある音声ファイルを選択します。

3. **要約の実行**:
   - 要約テンプレート（標準、ToDo特化など）を選択し、追加で指示したいことがあれば「追加指示」に入力します。
   - 「議事録を自動要約する」をタップします。

4. **結果の保存**:
   - 処理完了後、「要約された議事録」タブと「文字起こし」タブに自動で結果が振り分けられて表示されます。
   - 「コピー」ボタンでクリップボードにコピーするか、「保存」ボタンでファイルとして端末にダウンロードできます。

## スマホ（特にiPhone/iOS）での注意事項

- **マイク使用許可**: 
  - ブラウザからアクセスした際、マイクの使用許可を求めるポップアップが表示されます。「許可」を選択してください。
  - Safariで誤って拒否した場合は、iOSの設定アプリ ＞ `Safari` ＞ `マイク` から「確認」または「許可」に設定してください。
- **長時間の録音について**:
  - スマホの画面がスリープ状態になると、ブラウザの録音処理が途中で中断される場合があります。
  - 10分を超えるような長時間の会議では、スマホの標準録音アプリ（ボイスメモなど）であらかじめ録音し、生成されたM4AやMP3ファイルを本アプリに「ファイル選択」からアップロードして要約することをお勧めします。
- **ファイルサイズ制限**:
  - Webアップロードをスムーズに行うため、ファイルサイズは最大 **25MB** 程度までを推奨しています。
